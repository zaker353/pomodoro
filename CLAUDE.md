# 這個專案的說明

## 關於我
- 我不是工程師，看不懂程式術語，請一律用白話中文解釋。
- 每一步都幫我執行，並告訴我確切要做什麼、怎麼在瀏覽器打開來看結果。

## 這個專案是什麼
- 一個自用的「番茄鐘 + 白噪音」專注工具，不會上架、不需要帳號登入、不用架伺服器。
- 純離線 PWA：資料存在裝置本機（localStorage），沒網路也能用，手機可用「加入主畫面」變成像 App 一樣。
- 已上線：https://zaker353.github.io/pomodoro/ （GitHub 倉庫 zaker353/pomodoro，push 上去就會自動更新網站）。

## 主要功能
- 番茄鐘：可自訂時長、長休息、正計時、嚴格模式、預估番茄數。
- 白噪音混音器：24 種真實環境錄音（sounds/ 資料夾的 mp3），音檔讀不到時自動退回程式合成音。
- 任務清單、專注統計與熱力圖、徽章、週報、時段分析。
- 健康提醒（喝水/久坐/護眼/咖啡因/就寢）、呼吸練習、伸展引導、午睡、睡眠定時。
- 匯出/匯入備份 + GitHub 雲端全自動同步（開 App 自動下載合併、變更後 90 秒自動上傳）。

## 檔案結構（很簡單，別把它複雜化）
- `index.html` — 全部程式都在這一個檔案裡（HTML + CSS + JavaScript，約 2900 行），沒有用任何框架，改功能就是改這個檔。
- `sw.js` — 離線快取設定。裡面有 `CACHE = CACHE_PREFIX + "vXX"` 的版本號。
- `sounds/` — 24 個白噪音 mp3（出處和授權寫在 README.md，新增音效要一併補上出處）。
- `tests/test-pomo.js` — 120 項自動測試（用 jsdom 模擬瀏覽器）。
- `執行測試.bat` — 我自己雙擊就能跑測試；`開啟APP.bat` — 我自己雙擊就能打開 App。

## 固定規則
- 一次只改一件事，改完先讓我確認能用，再進行下一步。
- **絕對不能弄丟我的資料**（專注紀錄、任務、設定都在 localStorage）。「匯出備份／匯入」和「GitHub 雲端同步」功能一定要保留。
- 改完程式要跑自動測試：`npm test`（就是 tests/test-pomo.js）。新功能記得補測試進去。
- 測試通過後直接部署上線，不用問我：
  1. 先把 `sw.js` 的快取版本號 +1（例如 v12 → v13），不然手機上看不到新版。
  2. 如果有新增要離線用的檔案（例如新音效），記得加進 `sw.js` 的 FILES 清單。
  3. git commit 然後 push 到 GitHub，網站就會自動更新。
- 測不準、沒把握的改動就先別上線，跟我說明狀況。

## 特別注意
- `sw.js` 清快取時只能刪 `pomo-` 開頭的快取——同一個網域（zaker353.github.io）還有我其他 App（英文學習、塔羅牌），千萬不能把別人的離線快取清掉（以前出過這個 bug）。
- 雲端同步的三顆按鈕語意：合併＝兩邊資料都保留、覆蓋＝全部換掉，不要改掉這些白話說明。
